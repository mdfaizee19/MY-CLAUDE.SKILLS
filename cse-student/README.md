# CSE Student — Skills & Plugins

For coursework, DSA/competitive programming, lab assignments, group projects, and hackathon submissions.

## Skills included here (copied, ready to use)

All from the already-installed `superpowers` and `frontend-design` plugins, plus two standalone skills.

| Skill | Use it for |
|---|---|
| [`brainstorming`](skills/brainstorming/SKILL.md) | Scoping a project/assignment before writing code — nails down requirements first |
| [`systematic-debugging`](skills/systematic-debugging/SKILL.md) | Any bug or unexpected test failure — root-cause instead of guess-and-check |
| [`test-driven-development`](skills/test-driven-development/SKILL.md) | Writing tests before implementation for assignments with a spec/grader |
| [`writing-plans`](skills/writing-plans/SKILL.md) / [`executing-plans`](skills/executing-plans/SKILL.md) | Multi-step projects (final projects, multi-file labs) |
| [`using-git-worktrees`](skills/using-git-worktrees/SKILL.md) | Working on a feature branch without disturbing your main checkout |
| [`requesting-code-review`](skills/requesting-code-review/SKILL.md) / [`receiving-code-review`](skills/receiving-code-review/SKILL.md) | Peer review in group projects, or self-review before submitting |
| [`finishing-a-development-branch`](skills/finishing-a-development-branch/SKILL.md) | Wrapping up a project branch — merge/PR/cleanup decision |
| [`verification-before-completion`](skills/verification-before-completion/SKILL.md) | Before claiming an assignment "done" — run it, don't assume it |
| [`frontend-design`](skills/frontend-design/SKILL.md) | UI/UX for web-based course projects |
| [`hackathon-judge-readiness`](skills/hackathon-judge-readiness/SKILL.md) | Auditing a hackathon/demo-day submission against what judges actually notice |
| [`use-railway`](skills/use-railway/SKILL.md) | Deploying a course project or hackathon demo so it has a live URL |

## Built into Claude Code already (no file to copy, just use them)

- **`run`** — launch and screenshot your app to confirm a change actually works
- **`init`** — generate a CLAUDE.md for a new repo (good habit for every course project)
- **`dataviz`** — charts/plots for a data structures, ML, or stats assignment
- **`claude-api`** — reference if a project uses the Anthropic API directly
- **`security-review`** — worth running before submitting anything security-adjacent
- **`code-review`** / **`simplify`** / **`verify`** — general code quality passes

## Marketplace plugins worth installing (not yet on this machine)

See [`../marketplace-plugins/README.md`](../marketplace-plugins/README.md) for the full catalog. Most relevant to a CSE student:

- A **language-server plugin** matching your coursework language: `pyright-lsp` (Python), `clangd-lsp` (C/C++), `jdtls-lsp` (Java), `typescript-lsp`, `rust-analyzer-lsp`, `gopls-lsp`, `csharp-lsp`, `kotlin-lsp`, `ruby-lsp`, `php-lsp`, `lua-lsp`, `swift-lsp`
- `math-olympiad` — competition-style math/proof problems, useful for discrete math and algorithms courses
- `commit-commands` — cleaner git workflow for group projects
- `claude-md-management` — keep a repo's CLAUDE.md accurate as a project grows
- `learning-output-style` — Claude pauses at decision points and asks you to contribute code, good for actually learning the material instead of just getting an answer
- `explanatory-output-style` — Claude explains *why*, not just *what*, useful while studying
- `project-artifact` — generate a polished status page for a portfolio project
